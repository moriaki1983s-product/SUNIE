# SUNIE 利用ガイド（ざっくり版）

SUNIE は、「学び」と「意味づけ」をひとつの流れとして扱うための  
軽やかで柔軟な“学習インターフェース”です。

ここでは、SUNIE を使うときのイメージを  
**Client（学ぶ場所）** と **Server（考える場所）** の2章で紹介します。

---

# 1. Client（学ぶ場所）

Client は、SUNIE と人が出会う“入り口”です。  
Streamlit で動くシンプルな UI で、次のような使い方を想定しています。

## 1.1 直感的に触れる
- ブラウザを開くだけで SUNIE にアクセスできます  
- テキストを入力したり、ファイルをアップロードしたり  
- 「学習」「推論」「可視化」などのボタンを押すだけで動きます

## 1.2 学びの流れをつくる
- 学習したいデータを渡す  
- SUNIE が返す“意味のまとまり”を眺める  
- 必要なら再学習や追加データを投入する  

Client は、**SUNIE と対話しながら学びを深める場所**です。

## 1.3 教育現場での利用イメージ
- 生徒が自分の文章を読み込ませて構造を理解する  
- 教員が教材データを渡して意味のつながりを可視化する  
- 研究室でデータの特徴を素早く把握する  

難しい操作は必要ありません。  
Client は「SUNIE の窓」のような存在です。

---

# 2. Server（考える場所）

Server は、SUNIE の“頭脳”です。  
Flask で動く API と、必要に応じて Worker（Celery）が裏で動きます。

## 2.1 Client からの依頼を受け取る
- 「このデータを学習して」  
- 「この文章の意味構造を返して」  
- 「この処理を非同期で実行して」  

Server は、Client のリクエストを受け取り、  
**SUNIE のコアロジックを実行する役割**を担います。

## 2.2 意味構造の生成
Server の中心には、SUNIE の哲学に基づいた  
“意味のつながり”を扱う処理が組み込まれています。

- データを受け取る  
- 内部で構造化・整理する  
- 必要なら Worker に重い処理を渡す  
- 結果を Client に返す  

Server は、SUNIE の“静かなエンジン”です。

## 2.3 拡張しやすい構造
教育機関や研究室では、次のような拡張が可能です。

- 新しい分析モジュールの追加  
- 独自の学習アルゴリズムの組み込み  
- 外部データベースとの連携  
- Worker を増やして処理を高速化  

Server は、SUNIE の“成長する土台”として設計されています。

---

# おわりに

Client は **SUNIE と触れ合う場所**、  
Server は **SUNIE が考える場所**。

この2つがゆるやかにつながることで、  
SUNIE は「学び」と「意味づけ」を自然な流れとして扱えるようになります。

難しく考える必要はありません。  
SUNIE は、あなたの学びをそっと支えるために存在しています。
