# SUNIE / LUMIE  
教育機関向け 導入マニュアル（小・中・高・大学）

本書は、学校・教育機関のシステム担当者が  
**SUNIE / LUMIE を安全かつ確実に導入するための公式ガイド**です。

SUNIE / LUMIE は、学習・推論・意味構造を扱う新しい概念 OS であり、  
授業・研究・探究学習・AIリテラシー教育など幅広い用途に利用できます。

---

# 1. 導入の全体フロー

教育機関向けの標準導入手順は次の通りです。

1. **WSL2 の有効化（Windows）**  
2. **Ubuntu のインストール（WSL2 上）**  
3. **Docker Desktop の導入**  
4. **GitHub CLI の導入**  
5. **Python（開発者向け）**  
6. **SUNIE/LUMIE の取得と起動**

この順番で進めることで、  
**本番環境に近い Linux + Docker 構成を安全に再現**できます。

---

# 2. システム要件

## 2.1 ハードウェア
- CPU：Intel / AMD（仮想化支援 VT-x/AMD-V が有効）
- メモリ：8GB 以上推奨（最低 4GB）
- ストレージ：20GB 以上の空き容量
- OS：Windows 10/11（教育機関向けエディション対応）

## 2.2 ネットワーク要件
以下の外部サービスへのアクセスが必要です。

- GitHub（https://github.com）
- Docker Hub（https://hub.docker.com）
- Python パッケージ（https://pypi.org）
- Redis / RabbitMQ（Docker イメージ取得時）

プロキシ環境の場合は、担当者が事前に許可設定を行ってください。

## 2.3 管理者権限が必要な操作
- WSL2 の有効化  
- Docker Desktop のインストール  
- 仮想化機能（Hyper-V / Virtual Machine Platform）の有効化

# 11. トラブルシューティング

## 11.1 Docker が起動しない
- BIOS で仮想化支援が無効  
- Hyper-V が無効  
- 管理者権限不足  

## 11.2 GitHub にアクセスできない
- プロキシ設定が必要  
- 学校のフィルタリングでブロック  

## 11.3 Celery Worker が動かない
- Redis が起動していない  
- ポートが塞がれている

# 13. よくある質問（FAQ）

### Q. Windows だけで動かせますか？  
A. 基本的には WSL2 + Docker を推奨します。

### Q. Python を触らなくても使えますか？  
A. はい。Docker だけで利用可能です。

### Q. 学校 PC の制限が厳しい場合は？  
A. 管理者権限が必要な操作を事前に申請してください。
